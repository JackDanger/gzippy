## 0. Scope and baseline

Both kernels implement the same **late-discriminator, software-pipelined** shape: decode+spec-store on the **preloaded** litlen entry, consume, early next-index, refill-OR, litlen load (overlapping pos advance), dist preload, then `trailing < 256` в†’ hot back-edge.

| | gz `run_contig` | igzip `loop_block` |
|---|---|---|
| Hot literal loop | `asm_kernel.rs:537вЂ“705` | `igzip_decode_block_stateless.asm:507вЂ“556` |
| Litlen decode macro | inlined `588вЂ“634` | `decode_next_lit_len` `322вЂ“372`, invoked `515` |
| Refill + preload cadence | `662вЂ“698` (NIGHT19 в‰Ў igzip 524вЂ“552) | `524вЂ“552` |
| Copy | B3 scalar burst + MOVDQU `908вЂ“1053` | MOVDQU `603вЂ“627` |
| Discriminator | **one** `cmp {t5},256; jb 2b` `704вЂ“705` | **two** `cmp 256;je` `536вЂ“537` + `cmp 256;jl` `555вЂ“556` |

**GATED:** literal core в‰€ **37 ops/iter (gz)** vs **34вЂ“38 (igzip)** вЂ” at parity. **+2 instr/B** is not here. **+9% / +4вЂ“7% cyc/B** is **latency/recurrence + amortized backref**, not mispredict (gz IPC higher, br-miss lower).

## 1. THE RECURRENCE вЂ” op-by-op with latencies

### 1.1 Carried state (both)

| Register | gz | igzip | Role |
|---|---|---|---|
| Bit accumulator | `{bitbuf}` | `read_in` | Loop-carried bit buffer |
| Valid-bit count | `{bitsleft}` (saturated `|56`) | `read_in_length` (exact `lea`) | Feeds `SHLX` shift amount |
| Input cursor | `{pos}` / `{in_ptr}` | `next_in` | Loop-carried byte pointer |
| Preloaded entry | `{t1}` | `next_sym` | **Ends iter N, starts iter N+1** |
| Output | `{dst}` | `next_out` | Not on bit-recurrence |

The **throughput limiter** is the chain that must complete before the **next** litlen table load can issue:

```text
L1_load(entry[N]) в†’ extract_bc в†’ SHRX_consume в†’ AND_index в†’ L1_load(entry[N+1])
```

Std latencies (x86-64, L1-hit): ALU/LEA/SHRX/SHLX = **1 cyc**; L1 load = **4 cyc**; store = off hot back-edge recurrence.

### 1.2 igzip `loop_block` вЂ” one literal packet (short path, cntв‰Ґ1)

| Step | Asm | Ops | Lat | Deps on |
|------|-----|-----|-----|---------|
| **A0** | `540` (prev iter) | `mov next_sym, [state+lit+4*tmp3]` | **4** | index from prev iter |
| **A1** | `515` macro `332вЂ“371` | extract `rcx`=bc from `next_sym`; `and` mask | 1вЂ“2 | A0 |
| **A2** | macro `370вЂ“371` | `SHRX read_in`; `sub read_in_length, rcx` | 1+1 | A1 |
| **B1** | `520вЂ“521` | `lea`; `SHRX next_sym2` (trailing) | 1+1 | masked sym (в€Ґ A2) |
| **C1** | `524` | `mov tmp3, 0xFFF` | 0* | *off critical path |
| **C2** | `525` | `and tmp3, read_in` | 1 | **A2** (post-consume) |
| **D1** | `528вЂ“530` | `mov tmp1,[next_in]`; `SHLX`; `or read_in` | 4+1+1 | `next_in` (carried), `read_in_length` (carried) вЂ” **в€Ґ C2** |
| **E1** | `540` | `mov next_sym, [state+lit+4*tmp3]` | **4** | **C2** в†ђ **RECURRENCE TAIL** |
| **F1** | `543вЂ“547` | `mov/sub/shr/add next_in`; `lea read_in_length,[...+8*tmp1]` | 3вЂ“4 | `read_in_length` pre-refill вЂ” **scheduled during E1** |

**Critical path (cycles, ideal OOO):**

```text
A0(4) в†’ A1(1) в†’ A2(2) в†’ C2(1) в†’ E1_issue в†’ E1_complete(+4)  в‰€ 12 cycles/packet
         в””в”Ђ D1 overlaps C2..E1 (refill uses pre-OR read_in + carried read_in_length)
         в””в”Ђ F1 overlaps E1 load latency (NIGHT19/igzip-native pipelining)
```

**`read_in_length` tracking (igzip 543вЂ“547):**

```asm
mov  tmp1, 64
sub  tmp1, read_in_length      ; bytes to pull = (64-valid)>>3  (tmp1 в€Ґ E1)
shr  tmp1, 3
add  next_in, tmp1
lea  read_in_length, [read_in_length + 8*tmp1]   ; EXACT +8*bytes
```

3 ALU ops + 1 `lea`, all schedulable **during** the 4-cycle litlen load.

### 1.4 Head-to-head: is gzвЂ™s recurrence LONGER?

**GATED CONCLUSION: NO** вЂ” post-NIGHT19 (`asm_kernel.rs:635вЂ“657`, cites igzip `524вЂ“552`), the software-pipeline depth matches igzip. Remaining structural diffs:

| Mechanism | gz | igzip | Recurrence impact |
|---|---|---|---|
| **bitsleft / read_in_length** | `or {bitsleft},56` `683` вЂ” 1 uop, tops to в‰Ґ56 | `lea read_in_length,[...+8*tmp1]` `547` вЂ” exact | **gz в‰¤ igzip** on F1; both defer past E1 |
| **pos advance** | `679вЂ“682` identical 4-op chain | `543вЂ“546` | **Parity** |
| **Index extract** | const-mov then `and` `673вЂ“674` | `524вЂ“525` | **Parity** (NIGHT24) |
| **Discriminator** | 1Г— `cmp/jb` `704вЂ“705` | 2Г— `cmp` `536вЂ“537`, `555вЂ“556` | **gz shorter** |
| **p0 anchor** | `lea/sub` `583вЂ“584` every iter | none | **2 instr OFF recurrence** (NRK gate) |
| **Dist preload** | every iter `696вЂ“698` | every iter `550вЂ“552` | **Parity** (waste both pay) |

The **loop-carried recurrence** that caps literal throughput:


**Neither kernel can shave the 4-cycle L1 load** without prefetch or fusion. gz is not structurally longer on this chain; the **+cyc/B gap is not explained by a longer literal recurrence** (consistent with NRK: в€’4% instr, cyc TIE/REGRESS).

### 2.2 Mechanism (silesia, mean backref len в‰€ 6.3 B)

**igzip backref** (`603вЂ“627`): one `MOVDQU` + `sub repeat_length,16; jle loop_block` в‰€ **4вЂ“6 instr** per copy.

**gz B3** (`908вЂ“909` в†’ `70:`/`52:`): `cmp 40; jbe scalar` routes **в‰¤40 B** (almost all silesia backrefs) to **unconditional 5-word (40 B) burst** `987вЂ“996` в‰€ **~16 instr**, writing 40 B for ~6 B needed.

**Arithmetic (HYPOTHESIS, structural):**  
~19.6% / 41.2% в‰€ **0.47Г—** as many copy instr as literal instr в†’ copy adds **~2.0вЂ“2.5 instr/B** to the kernel total. Matches the measured +2.29 kernel surplus.

**NOT the surplus:**
- Literal discriminator (gz leaner)
- Per-block header (amortized tiny)
- CRC (outside `run_contig`, in the 10.3% bucket)
- `or bitsleft,56` vs exact `lea` (gz is fewer uops)

## 3. CAN gz BEAT igzip (cyc/B < igzip)?

**GATED FALSIFICATIONS (do not re-bet without new mechanism):**
- NRK / anchor removal в†’ instr в†“, cyc TIE (Intel) / REGRESS (AMD)
- RANK2 cursor #1 MOVDQU reroute for в‰¤40 в†’ **cyc/B REVERT** (+2вЂ“8% instr, monorepo +8.2% cyc)
- Incremental literal convergence в†’ at parity; no cyc win

**Structural gz advantages igzip lacks headroom to copy:**
1. **Single late discriminator** вЂ” igzip pays 2Г— `cmp 256` (`536вЂ“537`, `555вЂ“556`); gz one `cmp/jb` `704вЂ“705`
2. **Mask-once** `609` вЂ” igzip macro re-`and`s inside `341`
3. **Preloaded dist in `{dpre}`** вЂ” igzip re-loads at `575вЂ“577` on backref path; gz carries from `696вЂ“698`

**igzip advantages gz lacks:**
1. **MOVDQU-default copy** for short backrefs (`603вЂ“612`) вЂ” gz B3 scalar burst
2. **RFC dist table** `565` вЂ” single load vs gz in-register decode `832вЂ“841` (similar length; not the +2 i/B)
3. **Stateless** вЂ” no p0 anchor (gz pays 2 instr, **gated zero cyc**)

**HYPOTHESIS (unvalidated):** Beating igzip on **cyc/B** requires either (a) a **heroic T1-stateless monolith** with igzip-identical scheduling **and** copy default, or (b) a **recurrence-shortening** move that igzip cannot match (prefetch, 2-wide unroll) вЂ” not incremental instr shaving on the literal path.

## 4. Ranked moves вЂ” asm shape, hazards, differential, gate

Ranked **(1) recurrence-latency reduction**, **(2) instruction count**.

### MOVE 1 вЂ” L1 prefetch next litlen entry during consume в­ђ recurrence

**Mechanism:** Issue `prefetcht0 [ctx + index*4 + lit_off]` as soon as post-consume index is known (`674`), overlapping with refill `675вЂ“677`. Could hide all or part of the 4-cycle L1 latency on `{t1}` load `678`.

**Asm edit shape** (`asm_kernel.rs`, after `674`):

```asm
"and {t4:e}, {bitbuf:e}",
"prefetcht0 byte ptr [{ctx} + {t4}*4 + {lit_off}]",  // NEW
"mov {t2}, qword ptr [{in_ptr} + {pos}]",
```

**Byte-exact hazard:** None if prefetch is a pure hint (no cursor/state change).

**Differential:** `run_contig_ref_biased` unchanged; c2/c3 cursor equality; corpus sha==zcat.

**Gate:** Fulcrum paired cyc/B Intel+AMD, silesia+monorepo+nasa, Nв‰Ґ15. **Success:** median О”cyc/B < в€’spread on **both** arches. Prefetch can hurt (cache pollution) в†’ must measure.

**igzip headroom:** igzip could add the same prefetch at `525` вЂ” **not a durable advantage**, only a catch-up move.

### MOVE 2 вЂ” T1-stateless monolith (igzip `_04` transliteration) в­ђв­ђ heroic

**Mechanism:** Fresh `run_contig_t1_monolith`: no p0/d0/`85:` re-read, no RECLASS, EOB consumed (`T1-NRK-VERDICT` shape), **14 GP operands** (dedicated `{bc}` like `run_contig_stateless` `2540вЂ“2552`), copy tree = igzip `603вЂ“627` default (no B3), guards minimal.

**Asm edit shape:** New `#[naked]`/`asm!` block ~200 lines mirroring `loop_block:507вЂ“627` + gz table layout (`ctx` base). NRK is a partial prototype (`1186+` deletes `lea p0` but keeps B3 + resumable comments).

**Byte-exact hazard:** HIGH вЂ” every bail path, EOB cursor shape, subtable `91:` inline, copy overlap. Needs NRK-style ref lockstep + full corpus differential.

**Differential:** New `run_contig_t1_monolith_ref`; `GZIPPY_T1_MONOLITH=1` dispatch; sha==zcat all corpora.

**Gate:** cyc/B vs igzip, both arches, Nв‰Ґ15 interleaved. NRK showed **schedule sensitivity on AMD** вЂ” monolith must be scheduled from igzip objdump, not Rust `asm!` spill order.

**Why it might beat igzip:** gz-only discriminator + mask-once, **if** copy+schedule match igzip. **Why it might not:** NRK/reg-allocation regress on AMD.

### MOVE 4 вЂ” Dedicated `{bc}` register through consume (stateless pattern)

**Mechanism:** Like `run_contig_stateless` `2540вЂ“2552`: `bc` in `out(reg)` survives refill; consume `shrx {bitbuf},{bitbuf},{bc}`.

**Gate:** Already partially gated (stateless). **Not a recurrence win** вЂ” bc consumed before refill; `{t2}` reuse is intentional.

**Rank:** Low вЂ” instr-only, schedule risk.

### MOVE 5 вЂ” Skip dist preload on literal path (predicated)

**Mechanism:** Move `696вЂ“698` below `704` `jb 2b` fall-through only.

**Asm edit shape:** Cut dist preload from hot literal path; execute only on non-literal arms (`58:`, `21:`, cold `64:`).

**Byte-exact hazard:** **BREAKS** igzip convergence вЂ” dist preload hides L1 latency for the **next** backref after a literal run. After literal, next iter may be backref needing `{dpre}` вЂ” would need reload at `58:` (adds instr on backref, saves on literal).

**Recurrence:** Removes 1+4 cyc load from literal back-edge вЂ” **HYPOTHESIS (unvalidated)** ~3 instr + 4 cyc every literal iter. Silesia ~70% literals в†’ could matter.

**Gate:** cyc/B + backref-heavy corpus (monorepo).

**Risk:** Asymmetric win/loss; igzip keeps unconditional preload `550вЂ“552` for a reason (backref L1 hide).

### MOVE 6 вЂ” 2-packet software unroll (recurrence amortization)

**Mechanism:** Duplicate decode+consume+refill body; process 2 packets per guard check; one L1 miss amortized over 2 symbols.

**Asm edit shape:** Large вЂ” duplicate `588вЂ“705` block with mid-loop discriminator; second packet skips prologue preload.

**Byte-exact hazard:** EXTREME вЂ” E4 guards, cnt=2/3 packs, boundary exits, marker bails.

**Gate:** Corpus differential + cyc/B. High effort, high risk.

**Potential:** **Only move that shortens recurrence per output byte** without igzip parity вЂ” **HYPOTHESIS (unvalidated)**.

## 6. Strategic verdict (evidence-bound)

1. **Recurrence:** gz `run_contig` `662вЂ“705` is **the same depth** as igzip `524вЂ“556` post-NIGHT19. The binding chain is **`L1_load в†’ consume в†’ index в†’ L1_load` (~12 cyc/packet)**. gz is **not longer**; `or bitsleft,56` is в‰¤ igzipвЂ™s exact `lea`; gzвЂ™s discriminator is shorter. **p0 anchor is off-recurrence** (NRK gate).

2. **+2 instr/B:** **~19.6% scalar B3 burst** (`asm_kernel.rs:908вЂ“996` vs igzip `603вЂ“627`) is the dominant kernel surplus; literal loop is parity. Confirm with **AMD `perf annotate cycles:u`** on `c9417вЂ“c94f8`.

3. **Beat igzip?** Incremental literal edits are **exhausted** (NORTH-STAR BANK+ACCEPT). Remaining cyc/B paths: **(a)** igzip-faithful copy without B3 revert failure mode, **(b)** recurrence hacks (prefetch, unroll, dist-preload elision), **(c)** heroic monolith. gz already wins on discriminator/mask-once; igzip wins on **copy instr density**.

4. **Next gated experiment (cheapest high-signal):** MOVE 3 (igzip overlap dispatch + MOVDQU default, keep scalar only `>240`) on **AMD first** вЂ” instr-bound arch where RANK2 predicts payoff; gate nasa for B3 regression. In parallel, MOVE 1 (prefetch at `674`) is a 1-instruction probe with zero byte-exact risk.

**STRATEGIC FORK (user decision, from NORTH-STAR):** fund MOVE 2 monolith rewrite, or accept **~9% Intel / ~4вЂ“7% AMD** T1-vs-igzip floor as the incremental kernel ceiling.

# score/ — the gzippy parity scorecard

A frozen, greppable matrix of how gzippy's two production builds compare to the
`rapidgzip-native` ELF across **arch-os × threadcount × corpus**. Every number
is pasted verbatim from `scripts/fulcrum` on a frozen box. READMEs in this tree
contain ZERO hand-entered numbers — they are DERIVED from the one-line `SCORE:`
headers by `regen-index.sh`.

## HOW TO USE (reader agents start here)

1. **Read this file first** — the NORTH-STAR and DEAD-LEVERS below stop you
   re-deriving the map. The generated matrix + blindspots are at the bottom.
2. **A cell** lives at `score/<arch-os>/<threadcount>/<corpus>.md`. Its first
   line is a `SCORE:` header (grep it); the rest is yaml + prose + the verbatim
   fulcrum paste. Format is fixed by `SCHEMA.md`.
3. **PASS = ratio >= 0.99** where `ratio = rapidgzip-native wall / build wall`.
   Two builds are scored per cell: `gzippy-native` (pure-Rust, C-FFI off the
   decode graph) and `gzippy-isal` (ISA-L on the x86 decode path).
4. **To re-take a cell**: follow its `## RE-VERIFY` block (needs box access).
   Re-measuring archives the old file under `score/archive/...` (see SCHEMA.md).
5. **To rebuild this index**: `score/regen-index.sh` (greps SCORE lines, never
   re-measures). Edit only OUTSIDE the `<!-- ... GENERATED -->` markers.

## NORTH-STAR

gzippy must be at TIE-or-better (ratio >= 0.99 at EVERY threadcount) with
rapidgzip-native on every closable cell, with the **pure-Rust decoder as the
sole decode path and C-FFI off the decode graph**. The job is TRANSLITERATING
rapidgzip's architecture into Rust, not optimizing gzippy's wall in the
abstract. The scorecard exists to locate WHERE the pure-Rust (`native`) build
diverges from rapidgzip per cell, so convergence work is aimed, not guessed.

The decision the matrix gates: **can `gzippy-native` reach parity everywhere
(=> drop ISA-L), or is ISA-L the required x86 ship path on some arches/cells?**
intel (neurotic) is where native FAILS hardest; amd (solvency) is where native
WINS — the cross-arch contrast (Blindspot 3) is the fork.

## DEAD LEVERS — do NOT re-explore (banked, mechanism-backed)

- **u8-width marker ring rewrite (SegmentedU8).** DEAD. The marker ring is
  irreducibly u16 (MARKER_BASE=32768; markers ARE u16 values that cannot be
  stored u8) AND the clean bulk is ALREADY u8-direct. rapidgzip's
  `dataWithMarkers` is u16 too. There is no width knob. NORING's ~94ms is a
  FANTASY ceiling (markers MUST be stored u16; rg pays the same). The
  long-standing "u8 rewrite owed" mandate is SATISFIED/MOOT.
- **marker symbol-rate as a gz-vs-rg lever.** DEAD as an absolute. The 32%
  marker byte-fraction on silesia T8 is RG-SHARED (rg speculates too, ~34.5%
  marker fraction; calibration rg marker_emit 1,743M ≈ gz 2,234M). Only the
  DELTA is a lever.
- **cache / bandwidth / memory-bound.** DEAD. TMA shows memory_bound < 0.5%;
  the wall is FRONTEND/CORE-bound, not memory.

## LIVE LEVERS (where the DELTA is real — for context, not for this scorecard)

- **finalize op-reduction** — rg `cleanUnmarkedData`/finalize is ~5x lighter
  (rg 174M vs gz 924M ops); faithful port, wall ceiling ~20ms frozen solvency /
  ~60ms neurotic. Cheap, faithful, the next convergence step.
- **inner-loop frontend/bad-spec (engine-W)** — the TRUE native-parity ceiling
  (~51% of T1 wall; symbol-rate gap to ISA-L), asm-bounded ~0.667x; possibly
  unreachable without deeper-than-rung-c asm. This is the BIG, parity-determining
  bet — surfaced to the user as a fork.

## Boxes & comparator

| box      | arch-os       | access                                  | comparator ELF |
|----------|---------------|-----------------------------------------|----------------|
| neurotic | intel-x86_64  | `ssh -J 10.0.0.100 root@10.30.0.199`    | `/root/oracle_c/rapidgzip-native` |
| solvency | amd-x86_64    | `root@192.168.7.222`                    | `/root/gz-base/vendor/rapidgzip` build |

NEVER use the rapidgzip pip wheel as comparator. Freeze before measuring
(neurotic `/root/bench-lock.sh` or `no_turbo`; solvency `source
/root/bench-env.sh && freeze`). Mac excluded (no trustworthy wall).
